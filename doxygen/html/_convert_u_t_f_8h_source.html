00025 /* ---------------------------------------------------------------------
00026 
00027     Conversions between UTF32, UTF-16, and UTF-8.  Header file.
00028 
00029     Several funtions are included here, forming a complete set of
00030     conversions between the three formats.  UTF-7 is not included
00031     here, but is handled in a separate source file.
00032 
00033     Each of these routines takes pointers to input buffers and output
00034     buffers.  The input buffers are const.
00035 
00036     Each routine converts the text between *sourceStart and sourceEnd,
00037     putting the result into the buffer between *targetStart and
00038     targetEnd. Note: the end pointers are *after* the last item: e.g. 
00039     *(sourceEnd - 1) is the last item.
00040 
00041     The return result indicates whether the conversion was successful,
00042     and if not, whether the problem was in the source or target buffers.
00043     (Only the first encountered problem is indicated.)
00044 
00045     After the conversion, *sourceStart and *targetStart are both
00046     updated to point to the end of last text successfully converted in
00047     the respective buffers.
00048 
00049     Input parameters:
00050         sourceStart - pointer to a pointer to the source buffer.
00051                 The contents of this are modified on return so that
00052                 it points at the next thing to be converted.
00053         targetStart - similarly, pointer to pointer to the target buffer.
00054         sourceEnd, targetEnd - respectively pointers to the ends of the
00055                 two buffers, for overflow checking only.
00056 
00057     These conversion functions take a ConversionFlags argument. When this
00058     flag is set to strict, both irregular sequences and isolated surrogates
00059     will cause an error.  When the flag is set to lenient, both irregular
00060     sequences and isolated surrogates are converted.
00061 
00062     Whether the flag is strict or lenient, all illegal sequences will cause
00063     an error return. This includes sequences such as: <F4 90 80 80>, <C0 80>,
00064     or <A0> in UTF-8, and values above 0x10FFFF in UTF-32. Conformant code
00065     must check for illegal sequences.
00066 
00067     When the flag is set to lenient, characters over 0x10FFFF are converted
00068     to the replacement character; otherwise (when the flag is set to strict)
00069     they constitute an error.
00070 
00071     Output parameters:
00072         The value "sourceIllegal" is returned from some routines if the input
00073         sequence is malformed.  When "sourceIllegal" is returned, the source
00074         value will point to the illegal value that caused the problem. E.g.,
00075         in UTF-8 when a sequence is malformed, it points to the start of the
00076         malformed sequence.  
00077 
00078     Author: Mark E. Davis, 1994.
00079     Rev History: Rick McGowan, fixes & updates May 2001.
00080                  Fixes & updates, Sept 2001.
00081 
00082 ------------------------------------------------------------------------ */
